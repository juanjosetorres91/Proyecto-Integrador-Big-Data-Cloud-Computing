# Proyecto-Integrador-Big-Data-Cloud-Computing

Estructura base del proyecto:

```text
mi-proyecto/
├── data/               <- TODO lo relacionado con tus datos
│   ├── raw/            <- Los datos originales e inmutables (¡nunca se editan!)
│   ├── interim/        <- Datos intermedios que estás transformando
│   └── processed/      <- Datos limpios y finales listos para el análisis/modelado
│
├── notebooks/          <- Tus cuadernos de Jupyter (.ipynb) para exploración
│
├── src/                <- Código fuente (scripts de Python, R, etc.)
│
├── models/             <- Modelos entrenados guardados (archivos .pkl, .h5)
│
├── reports/            <- Reportes, presentaciones o análisis generados
│   └── figures/        <- Gráficos e imágenes generadas por tu código
│
├── docs/               <- Documentación adicional o manuales
│
├── .gitignore          <- Lista de archivos que GitHub DEBE ignorar
├── requirements.txt    <- Lista de librerías necesarias
└── README.md           <- La portada y explicación de tu proyecto
```
